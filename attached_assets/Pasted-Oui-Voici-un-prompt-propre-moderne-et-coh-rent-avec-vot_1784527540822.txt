### Refonte du système de chargement (Loading UX) — Interface moderne et cohérente avec le design

Nous souhaitons moderniser complètement le système de chargement de l'application afin d'obtenir une expérience utilisateur fluide, professionnelle et cohérente avec le futur design Figma du projet.

L'objectif est d'abandonner les anciens loaders génériques et d'adopter une stratégie moderne comparable aux applications SaaS professionnelles (Notion, GitHub, Google, Stripe, Linear, etc.).

Cette amélioration doit être purement visuelle et ergonomique : aucune logique métier ne doit être modifiée.

### 1. Principes généraux

Le système ne doit jamais donner l'impression d'être bloqué.

L'utilisateur doit toujours voir :

* qu'une action est en cours ;

* où il se trouve ;

* ce qui est en train d'être traité ;

* quand l'opération est terminée.

Tous les loaders doivent respecter l'identité visuelle du projet :

* mêmes couleurs ;

* mêmes rayons ;

* mêmes espacements ;

* mêmes animations ;

* même style que le design Figma.

Les animations doivent être légères, rapides et élégantes.

Éviter tout effet agressif ou trop visible.

### 2. Barre de progression de navigation

Lors d'un changement de page (Dashboard → Élèves, Élèves → Bulletins, etc.), afficher une fine barre de progression en haut de l'écran.

Comportement attendu :

* démarre immédiatement ;

* progresse de manière fluide ;

* disparaît automatiquement lorsque la page est prête.

Style :

* hauteur : 3 à 4 px ;

* coins arrondis ;

* couleur principale du système ;

* animation douce.

Ne pas bloquer l'interface pendant la navigation.

### 3. Skeleton Loaders

Pour toutes les pages contenant des données (tableaux, listes, cartes, statistiques, bulletins, etc.), remplacer les spinners par des Skeleton Loaders.

Exemples :

* cartes statistiques ;

* tableaux d'élèves ;

* tableaux d'enseignants ;

* listes de classes ;

* historique ;

* notifications ;

* portail parent.

Les skeletons doivent reproduire la structure réelle du contenu.

Animation :

* shimmer horizontal discret ;

* vitesse modérée ;

* compatible mode clair et sombre.

Le contenu doit apparaître progressivement sans clignotement.

### 4. Boutons avec état de chargement

Pour les actions rapides :

* connexion ;

* enregistrement ;

* suppression ;

* publication ;

* génération ;

* réinitialisation.

Le bouton doit afficher un loader intégré.

Exemple :

Publier → ⟳ Publication...

Règles :

* le bouton est désactivé pendant l'action ;

* la largeur du bouton ne doit pas changer ;

* l'icône tourne doucement ;

* le texte reste lisible.

Éviter les popups de chargement pour ces actions simples.

### 5. Overlay pour opérations longues

Pour les traitements importants :

* migration d'année scolaire ;

* génération massive de bulletins ;

* archivage ;

* sauvegarde ;

* recalcul des résultats ;

* import de données.

Afficher un overlay semi-transparent centré.

Contenu :

* logo de l'établissement ;

* spinner moderne ;

* titre ;

* description ;

* progression.

Exemple :

Migration en cours

Préparation des données...

████████░░░░░░ 52 %

Veuillez patienter.

Le fond doit rester visible mais non interactif.

### 6. Progression détaillée

Pour les traitements multi-étapes, afficher l'étape actuelle.

Exemple :

* Analyse des élèves

* Promotion des élèves

* Migration des classes

* Migration des enseignants

* Finalisation

L'utilisateur doit comprendre ce que fait le système.

### 7. Chargement du portail parent

Le portail parent doit avoir un chargement particulièrement soigné.

Afficher :

* logo ;

* nom de l'établissement ;

* skeleton des cartes ;

* skeleton des bulletins.

Ne jamais afficher un écran vide.

### 8. Installation PWA

Lors de l'installation de la PWA, prévoir un écran de préparation :

* logo ;

* nom de l'application ;

* animation légère ;

* message de préparation.

Style minimaliste et cohérent avec le design.

### 9. Notifications de fin d'action

Après une action réussie, afficher un feedback léger :

* ✓ Enregistré

* ✓ Résultats publiés

* ✓ Bulletin généré

* ✓ Année activée

Durée : 2 à 3 secondes.

Utiliser des toasts modernes en bas ou en haut de l'écran.

### 10. Responsive Design

Tous les loaders doivent fonctionner parfaitement sur :

* téléphone ;

* tablette ;

* ordinateur.

Sur mobile :

* taille réduite ;

* centrage parfait ;

* animations optimisées.

### 11. Accessibilité

Respecter les bonnes pratiques :

* contraste suffisant ;

* animations non agressives ;

* possibilité de réduire les animations si le système d'exploitation le demande (`prefers-reduced-motion`).

### 12. Performance

Les animations ne doivent pas dégrader les performances.

Utiliser :

* CSS transforms ;

* opacity ;

* animations GPU-friendly.

Éviter :

* reflows fréquents ;

* animations lourdes ;

* JavaScript inutile.

### 13. Cohérence visuelle

Créer un composant réutilisable pour chaque type de loader :

* `PageProgressBar`

* `SkeletonCard`

* `SkeletonTable`

* `LoadingButton`

* `LoadingOverlay`

* `ProgressOverlay`

* `ToastNotification`

Tous les modules du système devront utiliser ces composants afin de garantir une expérience uniforme.

### 14. Priorité d'intégration

Intégrer en priorité :

* Navbar (barre de progression)

* Tableau de bord

* Listes d'élèves

* Listes d'enseignants

* Portail parent

* Publication des résultats

* Migration d'année scolaire

* Génération des bulletins

* Connexion

### Objectif final

L'objectif est d'obtenir une expérience de chargement propre, fluide, élégante et cohérente avec le design Figma, donnant l'impression d'une application SaaS moderne et haut de gamme.

L'utilisateur ne doit jamais voir :

* une page blanche ;

* un blocage apparent ;

* un spinner générique sans contexte.

Chaque chargement doit être :

* informatif ;

* esthétique ;

* rapide à percevoir ;

* parfaitement intégré à l'identité visuelle du système.

Cette amélioration doit renforcer la sensation de qualité, de performance et de professionnalisme de toute la plateforme, sans modifier la logique métier existante.

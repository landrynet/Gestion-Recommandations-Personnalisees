Phase 2 : Gestion des comptes utilisateurs et authentification
1. Création d'un compte enseignant

Le préfet est le seul autorisé à créer les comptes des enseignants.

Lors de la création, il renseigne :

Nom
Postnom
Prénom
Sexe
Téléphone
Adresse e-mail (identifiant de connexion)
Fonction
Matières et classes (affectées ultérieurement si nécessaire)

Le système génère automatiquement :

un identifiant unique (UUID ou ID interne) ;
un mot de passe temporaire sécurisé et aléatoire (par exemple 12 caractères).

Le compte est créé avec le statut :

Première connexion obligatoire

2. Première connexion

L'enseignant se connecte avec :

son adresse e-mail ;
le mot de passe temporaire fourni par le préfet.

Le système détecte qu'il s'agit de la première connexion.

Avant d'accéder au tableau de bord, il est obligé de :

saisir le mot de passe temporaire ;
définir un nouveau mot de passe ;
confirmer ce nouveau mot de passe.

Tant que cette étape n'est pas terminée, aucun accès n'est autorisé.

3. Politique de mot de passe

Le nouveau mot de passe doit respecter des règles minimales.

Exemple :

au moins 8 ou 10 caractères ;
une lettre majuscule ;
une lettre minuscule ;
un chiffre ;
un caractère spécial.

Le système affiche le niveau de sécurité :

Faible
Moyen
Fort

Seul un mot de passe Fort est accepté.

4. Connexions suivantes

Une fois le mot de passe changé :

L'utilisateur se connecte simplement avec :

son adresse e-mail ;
son nouveau mot de passe.

Le mot de passe temporaire devient définitivement invalide.

5. Réinitialisation

Si l'utilisateur oublie son mot de passe :

Le préfet peut :

générer un nouveau mot de passe temporaire ;
réinitialiser le compte.

Au prochain accès, l'utilisateur devra de nouveau choisir un mot de passe personnel.

6. Compte du préfet

Le compte du préfet est créé lors de l'installation initiale.

Il possède lui aussi :

un mot de passe temporaire.

À sa première connexion :

il est obligé de le modifier ;
il doit choisir un mot de passe fort.
7. Évolutions futures

Prévoir dès maintenant une architecture compatible avec :

authentification à deux facteurs (2FA) ;
code de vérification par e-mail ;
application d'authentification (Google Authenticator, Microsoft Authenticator, etc.) ;
récupération sécurisée du mot de passe ;
journal des connexions ;
détection des connexions suspectes.

Je vous conseille également de ne jamais utiliser un nom d'utilisateur. Utilisez uniquement l'adresse e-mail comme identifiant de connexion. C'est plus simple pour les utilisateurs, plus professionnel et plus facile à intégrer avec des fonctionnalités futures comme la réinitialisation de mot de passe ou la double authentification.

Ajustements finaux du système scolaire

Le système est désormais fonctionnel. Nous sommes dans la phase des derniers ajustements afin d'obtenir une version stable, cohérente et professionnelle.

Merci de vérifier et de corriger les points suivants.

1. Portail parent
Le portail parent ne doit jamais recalculer les moyennes, les pourcentages ou les classements.
Toutes les informations affichées doivent provenir des données officielles enregistrées sur le bulletin.
Si une période n'a pas encore été publiée, aucune donnée concernant cette période ne doit être affichée.
La moyenne générale et la place générale ne doivent être affichées que lorsqu'elles sont réellement disponibles.
2. Publication des bulletins
Lorsqu'un préfet publie une période, seules les données de cette période deviennent visibles dans le portail parent.
Il est interdit d'utiliser le pourcentage d'une période comme moyenne générale.
Le portail doit toujours rester cohérent avec le bulletin officiel.
3. Génération des bulletins
Adapter le bulletin au modèle officiel que je vais fournir.
Ajouter clairement les informations de chaque période (pourcentage, moyenne, classement).
La présentation doit être plus moderne, professionnelle et parfaitement lisible.
4. Progressive Web App (PWA)
Si la PWA est installée depuis le portail parent (via le QR Code), elle doit toujours ouvrir directement le portail parent.
Elle ne doit jamais rediriger le parent vers la page de connexion du personnel.
Vérifier le routage, le start_url et toutes les redirections afin que ce comportement soit garanti.
5. Performances
Vérifier que toutes les pages utilisent les indicateurs de chargement lorsque des données sont récupérées.
Optimiser les requêtes afin d'éviter les temps d'attente inutiles.
Supprimer les éventuels chargements bloquants.
6. Interface utilisateur
Uniformiser les couleurs, les icônes et les boutons.
Vérifier l'affichage sur téléphone, tablette et ordinateur.
Corriger les problèmes d'alignement, de responsive design et d'espacement.
7. Contrôles finaux

Avant de considérer le travail terminé, vérifier que :

aucune erreur JavaScript n'apparaît dans la console ;
aucune erreur serveur n'est générée ;
toutes les pages sont accessibles selon les droits de chaque utilisateur ;
les parents ne voient que leurs propres données ;
les enseignants et préfets ne voient que les fonctionnalités correspondant à leur rôle.

L'objectif est d'obtenir une version stable, cohérente et prête pour un déploiement en production, avec une expérience utilisateur fluide et des données parfaitement conformes aux bulletins officiels.

Cette idée est excellente. En fait, c'est ainsi que fonctionnent beaucoup de logiciels scolaires professionnels. **L'enseignant ne doit travailler que sur la période ouverte**, et non voir toute l'année scolaire.

Cela évite les erreurs, les modifications accidentelles et simplifie énormément l'interface.

Je structurerais la fonctionnalité ainsi :

---

# Gestion des périodes actives pour les enseignants

L'objectif est de limiter le travail des enseignants uniquement à la période scolaire actuellement ouverte par le préfet.

Le préfet devient le seul responsable de l'ouverture, de la fermeture et de l'archivage des périodes.

## Fonctionnement

Une année scolaire est composée de plusieurs périodes.

Exemple :

* Première période
* Deuxième période
* Troisième période
* Quatrième période (si nécessaire)

À un instant donné, **une seule période est active** pour les enseignants.

---

## Activation par le préfet

Depuis le module **Gestion des périodes**, le préfet peut :

* ouvrir une période ;
* fermer une période ;
* publier les résultats de cette période ;
* archiver automatiquement la période après publication ;
* activer la période suivante.

Une seule période peut être ouverte à la fois.

Le système doit empêcher l'ouverture simultanée de plusieurs périodes, sauf si cette option est explicitement prévue dans les paramètres.

---

## Interface enseignant

Lorsqu'un enseignant se connecte :

Il ne doit voir **que la période active**.

Exemple :

Si la Première période est ouverte :

L'enseignant voit uniquement :

* les saisies de notes de la Première période ;
* les évaluations de cette période ;
* les moyennes provisoires de cette période.

Les autres périodes restent invisibles.

L'enseignant ne peut ni les consulter, ni les modifier.

L'interface devient plus simple et évite toute confusion.

---

## Publication

Lorsque le préfet publie officiellement les résultats d'une période :

Cette période devient automatiquement :

**Archivée côté enseignant.**

L'enseignant ne peut plus :

* modifier les notes ;
* ajouter des évaluations ;
* supprimer des données.

Toutes les données deviennent accessibles uniquement en lecture si les permissions le permettent.

---

## Activation de la période suivante

Après l'archivage :

Le préfet active la période suivante.

Exemple :

Première période → Archivée

Deuxième période → Active

Tous les enseignants voient automatiquement uniquement la Deuxième période.

Aucune manipulation n'est nécessaire de leur part.

---

## Nouvelle année scolaire

Lorsqu'une nouvelle année scolaire est activée :

Toutes les périodes de l'année précédente restent archivées.

Les enseignants ne voient plus les anciennes périodes par défaut.

Ils commencent directement avec la **Première période de la nouvelle année scolaire**, une fois celle-ci ouverte par le préfet.

Les anciennes années restent consultables uniquement dans un espace **Archives**, si les permissions le permettent.

---

## Archives

Créer un espace **Archives** permettant de consulter :

* les anciennes années scolaires ;
* les anciennes périodes ;
* les notes publiées ;
* les bulletins générés.

Ces archives sont accessibles uniquement en lecture.

Aucune modification ne doit être possible.

---

## Sécurité

Le système doit empêcher toute modification :

* d'une période publiée ;
* d'une période archivée ;
* d'une année scolaire clôturée.

Toutes ces données deviennent définitivement en lecture seule.

---

## Synchronisation avec le portail parent

Le portail parent ne doit afficher que les périodes officiellement publiées par le préfet.

Une période non publiée ne doit jamais être visible.

Lorsqu'une période est archivée après publication, ses résultats restent accessibles dans l'historique du portail parent.

---

## Objectif

Créer une gestion des périodes **simple, sécurisée et progressive**. Les enseignants travaillent uniquement sur la période active, ce qui réduit les risques d'erreur et allège l'interface. Une fois les résultats publiés, la période est automatiquement verrouillée et archivée. Le préfet contrôle entièrement le cycle de vie des périodes (ouverture, publication, archivage et activation de la suivante), tandis que les parents accèdent uniquement aux périodes officiellement publiées. Cette logique s'applique également aux nouvelles années scolaires, garantissant une continuité, une traçabilité et une parfaite cohérence des données.
